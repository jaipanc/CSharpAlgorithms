39. Combination Sum

Problem Summary

Given:

  • int[] candidates (distinct numbers)

  • int target

Return all unique combinations where numbers can be chosen unlimited times and sum equals target.

Example:

candidates = [2,3,6,7] target = 7

Expected:

[[2,2,3], [7]]

🌳 Backtracking Tree Trace

We build combinations using DFS (backtracking).

Rules:

  • We can reuse the same number.

  • We move forward in the array (avoid duplicates).

  • Stop when sum > target.

  • If sum == target → add to result.

Example: candidates = [2,3,6,7], target = 7

Start:

[] target = 7

Step 1 — Choose 2

[] └── [2] (remaining 5)

Choose 2 again

[] └── [2] └── [2,2] (remaining 3)

Choose 2 one more time

[] └── [2] └── [2,2] └── [2,2,2] (remaining 1)

1 < 2 → ❌ STOP (backtrack)

From [2,2], try 3

[] └── [2] └── [2,2] └── [2,2,3] (remaining 0)

✅ VALID → Add to result

Backtrack.

From [2,2], try 6

Remaining = 3 - 6 = negative ❌

From [2], try 3

[] └── [2] └── [2,3] (remaining 2)

Try 3 again:

[2,3,3] → remaining -1 ❌

Try 6:

[2,3,6] → negative ❌

Backtrack.

From root, try 3

[] └── [3] (remaining 4)

Try 3 again:

[3,3] (remaining 1)

1 < 3 → ❌

Backtrack.

From root, try 6

[6] (remaining 1) ❌

From root, try 7

[7] (remaining 0)

✅ VALID → Add

✅ Final Result

[[2,2,3], [7]]

🧠 Why We Pass startIndex

When we recurse:

  • We pass current index i

  • So we can reuse same number

  • But we never go backwards (avoid duplicates)

💻 C# Implementation

public class Solution { public IList<IList<int>> CombinationSum(int[] candidates, int target) { var result = new List<IList<int>>(); Backtrack(candidates, target, 0, new List<int>(), result); return result; } private void Backtrack(int[] candidates, int remaining, int start, List<int> current, List<IList<int>> result) { if (remaining == 0) { result.Add(new List<int>(current)); return; } if (remaining < 0) return; for (int i = start; i < candidates.Length; i++) { current.Add(candidates[i]); // allow reuse -> pass i (not i+1) Backtrack(candidates, remaining - candidates[i], i, current, result); current.RemoveAt(current.Count - 1); // backtrack } } }

⏱ Time & Space Complexity

Time Complexity: Exponential → O(2^target)

Space Complexity: O(target) recursion depth
