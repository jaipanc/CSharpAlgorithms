78.Subsets

Problem Statement

Given an integer array nums of unique elements, return all possible subsets (the power set). The solution set must not contain duplicate subsets.

Example

Input: nums = [1,2,3]
Output: [[],[1],[2],[1,2],[3],[1,3],[2,3],[1,2,3]]

Approach 1: Backtracking (Recursive)

Visual Step-by-Step Process

Let's trace through nums = [1,2,3]:

_ [] / \ [1] [] / \ / \ [1,2] [1] [2] [] / | / | | \ | \ [1,2,3][1,2][1,3][1][2,3][2][3][]

Decision Tree Visualization:

Level 0: Start with empty subset [] ↓ Level 1: Include/exclude 1 → [1], [] ↓ Level 2: Include/exclude 2 → [1,2], [1], [2], [] ↓ Level 3: Include/exclude 3 → [1,2,3], [1,2], [1,3], [1], [2,3], [2], [3], []

Each level represents a decision about whether to include the current element or not.

Implementation in C#

public static IList<IList<int>> Subsets(int[] nums) { IList<IList<int>> result = []; Backtrack(0, []); return result; void Backtrack(int index, List<int> curr) { result.Add([.. curr]); // add subset at every step for (int i = index; i < nums.Length; i++) { if (i > index && nums[i] == nums[i - 1]) continue; // skip duplicates curr.Add(nums[i]); Backtrack(i + 1, curr); curr.RemoveAt(curr.Count - 1); } } }

Time and Space Complexity

  • Time Complexity: O(2ⁿ × n) where n is the length of nums

    • There are 2ⁿ subsets to generate

    • Each subset takes O(n) time to copy

  • Space Complexity: O(2ⁿ × n) for storing all subsets

    • Recursive approach also uses O(n) space for recursion stack

The backtracking approach is most intuitive and commonly used in interviews!
