17. Letter Combinations of a Phone Number

Problem Understanding

Given a string containing digits from 2-9 inclusive, return all possible letter combinations that the number could represent based on traditional telephone button mappings.

Example:

  • Input: "23"

  • Output: ["ad", "ae", "af", "bd", "be", "bf", "cd", "ce", "cf"]

Step-by-Step Implementation

Step 1: Set up the digit-to-letter mapping

using System; using System.Collections.Generic; public class Solution { public IList<string> LetterCombinations(string digits) { // Handle edge case - empty input if (string.IsNullOrEmpty(digits)) return new List<string>(); // Create mapping for telephone keypad Dictionary<char, string> phoneMap = new Dictionary<char, string> { {'2', "abc"}, {'3', "def"}, {'4', "ghi"}, {'5', "jkl"}, {'6', "mno"}, {'7', "pqrs"}, {'8', "tuv"}, {'9', "wxyz"} };

Step 2: Use backtracking approach

List<string> result = new List<string>(); Backtrack(digits, 0, "", phoneMap, result); return result; } private void Backtrack(string digits, int index, string currentCombination, Dictionary<char, string> phoneMap, List<string> result) { // Base case: if we've processed all digits if (index == digits.Length) { result.Add(currentCombination); return; } // Get the letters corresponding to current digit char digit = digits[index]; string letters = phoneMap[digit]; // Try each letter for the current digit foreach (char letter in letters) { // Add current letter to combination and move to next digit Backtrack(digits, index + 1, currentCombination + letter, phoneMap, result); } } }

Complete Solution with Explanation

using System; using System.Collections.Generic; public class Solution { public IList<string> LetterCombinations(string digits) { // Edge case: empty input returns empty list if (string.IsNullOrEmpty(digits)) return new List<string>(); // Mapping of digits to letters (telephone keypad style) Dictionary<char, string> phoneMap = new Dictionary<char, string> { {'2', "abc"}, // Digit 2 maps to letters a, b, c {'3', "def"}, // Digit 3 maps to letters d, e, f {'4', "ghi"}, // And so on... {'5', "jkl"}, {'6', "mno"}, {'7', "pqrs"}, {'8', "tuv"}, {'9', "wxyz"} }; List<string> result = new List<string>(); // Start the backtracking process Backtrack(digits, 0, "", phoneMap, result); return result; } private void Backtrack(string digits, int index, string currentCombination, Dictionary<char, string> phoneMap, List<string> result) { // Base case: when we've processed all digits // This means we have a complete combination if (index == digits.Length) { result.Add(currentCombination); return; } // Get the current digit we're processing char digit = digits[index]; // Get all possible letters for this digit string letters = phoneMap[digit]; // Try each letter for the current position foreach (char letter in letters) { // Recursive call: // - Move to next digit (index + 1) // - Add current letter to our combination // - Continue building the combination Backtrack(digits, index + 1, currentCombination + letter, phoneMap, result); } } }

How It Works - Trace Example

Let's trace through digits = "23":

Initial call: Backtrack("23", 0, "", map, result) Step 1: index=0, digit='2', letters="abc" ├── letter='a': Backtrack("23", 1, "a", map, result) │ ├── index=1, digit='3', letters="def" │ ├── letter='d': Backtrack("23", 2, "ad", map, result) → index==2, add "ad" │ ├── letter='e': Backtrack("23", 2, "ae", map, result) → index==2, add "ae" │ └── letter='f': Backtrack("23", 2, "af", map, result) → index==2, add "af" ├── letter='b': Backtrack("23", 1, "b", map, result) │ ├── letter='d': add "bd" │ ├── letter='e': add "be" │ └── letter='f': add "bf" └── letter='c': Backtrack("23", 1, "c", map, result) ├── letter='d': add "cd" ├── letter='e': add "ce" └── letter='f': add "cf"

Alternative Iterative Approach

public class Solution { public IList<string> LetterCombinations(string digits) { if (string.IsNullOrEmpty(digits)) return new List<string>(); Dictionary<char, string> phoneMap = new Dictionary<char, string> { {'2', "abc"}, {'3', "def"}, {'4', "ghi"}, {'5', "jkl"}, {'6', "mno"}, {'7', "pqrs"}, {'8', "tuv"}, {'9', "wxyz"} }; List<string> result = new List<string> { "" }; foreach (char digit in digits) { string letters = phoneMap[digit]; List<string> temp = new List<string>(); foreach (string combination in result) { foreach (char letter in letters) { temp.Add(combination + letter); } } result = temp; } return result; } }

Key Points Explained

  1. Base Case: When we've processed all digits (index == digits.Length), we have a complete combination

  2. Recursive Case: For each digit, try every possible letter and recursively process remaining digits

  3. Backtracking: The recursion naturally handles backtracking by exploring all possibilities

  4. Time Complexity: O(3N × 4M) where N is digits with 3 letters, M with 4 letters

  5. Space Complexity: O(3N × 4M) for output + O(N) for recursion stack

The solution efficiently generates all possible combinations using the fundamental principle of backtracking - explore all choices at each step while maintaining the current state.
