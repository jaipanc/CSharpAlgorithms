🌳 Depth-First Search (DFS)

Before you touch DFS, you should lock these prerequisites. If even one is shaky, DFS will feel “magical” and confusing instead of logical.

🔑 Absolute MUST prerequisites for DFS

1️⃣ Recursion fundamentals (NON-NEGOTIABLE)

DFS is recursion in disguise.

You must be comfortable with:

  • Base case

  • Recursive call

  • Call stack behavior

  • Backtracking (undoing state)

You should instantly understand this:

void Foo(int x) { if (x == 0) return; // base case Foo(x - 1); // recursive call }

If recursion feels unclear → stop and fix this first.

2️⃣ Stack concept (implicit vs explicit)

DFS uses a stack:

  • Recursive DFS → call stack

  • Iterative DFS → Stack<T>

You must know:

  • LIFO behavior

  • push / pop

  • how stack depth maps to recursion depth

Mental model

DFS recursion ≡ Stack push Returning ≡ Stack pop

3️⃣ Graph representation (very important)

DFS never works on “raw data” — it works on connections.

You must understand:

Adjacency list (most common)

Dictionary<int, List<int>> graph

Example:

1 → 2, 3 2 → 4 3 → -

This is the language of DFS.

4️⃣ Visited tracking (to avoid infinite loops)

Especially for graphs.

You must know:

HashSet<int> visited

Why?

  • Prevent cycles

  • Prevent repeated work

  • Control correctness

If you don’t understand why visited is needed → DFS will break.

5️⃣ Tree vs Graph difference (critical)

Many beginners confuse this.

Structure

Visited needed?

Tree

❌ No (no cycles)

Graph

✅ Yes

If you mix this up → bugs.

6️⃣ Loop basics (for neighbors)

DFS always expands neighbors.

You must be comfortable with:

foreach (var nei in graph[node]) { // DFS(nei) }

If loops feel mechanical or unclear, DFS will feel chaotic.

7️⃣ Boundary checks (grid DFS prerequisite)

For matrix / island problems, you must be fluent with:

  • Row / column indexing

  • Bounds checking

if (r < 0 || c < 0 || r >= rows || c >= cols) return;

This is mandatory before grid DFS.

🧠 Optional but STRONGLY recommended

8️⃣ Dry-run recursion on paper

You should be able to manually trace:

DFS(1) └─ DFS(2) └─ DFS(4) └─ DFS(3)

If you can’t dry-run → you don’t understand DFS yet.

9️⃣ Basic problem patterns

You should know these before DFS:

  • Linear scan

  • Nested loops

  • Simple condition checks

DFS combines all three.

🚫 What you do NOT need before DFS

❌ Dynamic Programming ❌ Greedy algorithms ❌ Advanced graph theory ❌ Math-heavy concepts

DFS is foundational, not advanced.

✅ Readiness checklist (be honest)

You’re ready for DFS if you can say YES to all:

  • [ ] I understand recursion without memorizing

  • [ ] I know how a stack works

  • [ ] I can represent a graph as adjacency list

  • [ ] I understand why visited is needed

  • [ ] I can dry-run recursive calls

  • [ ] I can do boundary checks in a matrix

If even one ❌, fix that first.

🛣️ Suggested order BEFORE DFS (1–2 days max)

  1. Recursion basics

  2. Stack (LIFO)

  3. Graph representation

  4. Tree traversal (pre/in/post order)

  5. Grid boundaries
