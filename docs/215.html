215.Kth Largest Element in an Array

Problem

Given nums and k, return the kth largest element.

Example:

  • nums = [3,2,1,5,6,4], k = 2

  • Sorted descending: [6,5,4,3,2,1]

  • 2nd largest = 5

Best DSA approach: Min-Heap of size k

Why min-heap?

We only care about the top k largest numbers. So we maintain a min-heap containing those top k.

  • The root (smallest in heap) is the kth largest overall

Invariant (the rule we maintain)

After processing elements so far:

  • heap contains the k largest elements seen so far

  • heap size is at most k

  • Peek() is the current kth largest

Step-by-step example (visual)

nums = [3,2,1,5,6,4], k=2

We keep a min-heap of size 2.

  1. Add 3 → heap: [3]

  2. Add 2 → heap: [2,3] (min is 2)

  3. Add 1 → heap: [1,3,2] size > 2 → remove min (1) → heap: [2,3]

  4. Add 5 → heap: [2,3,5] size > 2 → remove min (2) → heap: [3,5]

  5. Add 6 → heap: [3,5,6] size > 2 → remove min (3) → heap: [5,6]

  6. Add 4 → heap: [4,6,5] size > 2 → remove min (4) → heap: [5,6]

Now Peek() = 5 ✅ which is the 2nd largest.

C# Solution (Min-Heap using PriorityQueue)

using System.Collections.Generic; public class Solution { public int FindKthLargest(int[] nums, int k) { // Min-heap: smallest priority comes out first var heap = new PriorityQueue<int, int>(); // element, priority foreach (int num in nums) { heap.Enqueue(num, num); // push current number if (heap.Count > k) // keep only k elements heap.Dequeue(); // remove smallest } // The smallest among the k kept elements is the kth largest overall return heap.Peek(); } }

Why Peek() is the answer

At the end, heap holds the k largest elements. The smallest inside that heap is exactly the kth largest.

Example for k=2: Heap ends as [5,6] Smallest is 5 → that’s the 2nd largest.

Complexity

  • Time: O(n log k) (each insert/remove is log k, not log n)

  • Space: O(k)

Common mistakes

  • ❌ Using max-heap of all elements → O(n log n)

  • ❌ Sorting the whole array → O(n log n)

  • ❌ Returning Dequeue() at the end (that would remove it) — Peek() is cleaner for the final value
